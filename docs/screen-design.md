# ナレッジ管理アプリ 画面設計書

## 1. 共通レイアウト

### 1.1 ヘッダー
```
[ロゴ] ナレッジ管理システム                    [ユーザー名] [ログアウト]
----------------------------------------------------------------
[FAQ検索] [投稿] [承認待ち] [管理]                [重要な通知エリア]
```

### 1.2 ナビゲーション（役割別表示）
- **オペレータ**: FAQ検索、投稿、マイ投稿
- **SV**: FAQ検索、投稿、マイ投稿、承認待ち、承認済み管理
- **Admin**: 全機能 + ユーザー管理、承認グループ管理

## 7. レスポンシブ対応

### 7.1 モバイル表示時の調整
- 検索条件をアコーディオン形式に
- リスト表示を縦積みレイアウトに
- ボタンサイズを指での操作に適したサイズに

### 7.2 タブレット表示時の調整
- サイドナビゲーション表示
- 一覧と詳細の2ペイン表示

## 8. 画面遷移フロー

```
ログイン → FAQ検索画面（デフォルト）
    ↓
FAQ検索画面 ⟷ FAQ詳細画面
    ↓
FAQ投稿画面 ⟷ プレビュー画面
    ↓
マイ投稿一覧 ⟷ 編集画面
    ↓
承認待ち一覧（SV） ⟷ 承認画面
    ↓
管理画面（Admin）
```

## 9. UI/UX設計指針

### 9.1 カラーパレット
- **プライマリ**: 青系（#2563eb）
- **セカンダリ**: グレー系（#6b7280）
- **重要**: 赤系（#dc2626）
- **成功**: 緑系（#16a34a）
- **警告**: 黄系（#eab308）

### 9.2 アイコン使用
- 🔴 重要マーク
- 📝 編集
- ✅ 承認
- ❌ 却下/削除
- 🔍 検索
- ➕ 新規追加

### 9.3 操作性重視
- 大きめのクリック領域
- 明確な状態表示
- 直感的なナビゲーション
- 電話対応中でも操作しやすいシンプルなUI
